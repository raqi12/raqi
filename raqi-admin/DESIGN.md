# Arabic Enterprise Dashboard Design System

## Project Vision

This application is an Arabic-first enterprise platform.

The design must feel modern, premium, trustworthy, and highly professional.

The dashboard should look comparable in quality to:

* Linear
* Stripe Dashboard
* Vercel
* Notion
* GitHub
* Supabase

However, the interface must be optimized for Arabic users and RTL layouts from the beginning.

The design should feel like a product used daily by companies, administrators, managers, finance teams, operations teams, and decision makers.

---

# Language Requirements

## Arabic First

Arabic is the primary language.

All layouts must be designed for RTL first.

The design should not feel like an English dashboard that was translated into Arabic.

Arabic must be the default experience.

---

## RTL Requirements

All pages must support:

* RTL layouts
* RTL navigation
* RTL tables
* RTL forms
* RTL modals
* RTL dropdowns
* RTL charts
* RTL breadcrumbs

Text alignment:

Arabic:
Right aligned

English:
Left aligned

Numbers:
Display naturally inside Arabic content.

---

# Design Philosophy

## Simplicity

Every component should have a clear purpose.

Avoid visual noise.

Avoid decorative UI.

Focus on usability.

---

## Professionalism

The dashboard should feel:

* Premium
* Enterprise-grade
* Trustworthy
* Fast
* Organized

Not like a dashboard template.

---

## Clarity

Users should understand every page immediately.

Information hierarchy must be obvious.

The most important data should be visible first.

---

## Consistency

Consistency is more important than creativity.

Use the same:

* spacing
* typography
* colors
* button styles
* card styles
* table patterns

throughout the application.

---

# Final Audit Checklist

Before completing any design work:

1. Is the page optimized for Arabic users?
2. Does the page support RTL correctly?
3. Is the hierarchy clear?
4. Is the spacing consistent?
5. Is the typography professional?
6. Is the UI responsive?
7. Is dark mode supported?
8. Is accessibility respected?
9. Does it feel enterprise-grade?
10. Would this quality be acceptable in Stripe, Linear, or Vercel?

If the answer to any question is no, continue improving the design.
